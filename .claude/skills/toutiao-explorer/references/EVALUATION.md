# 收益成本分析

评估活动的投入产出比，优化活动选择策略。

## 收益评估

### 现金收益计算

```
净收益 = 预期收益 - 完成成本
```

**现金激励活动**：
- 固定奖金：直接记录金额
- 阅读量奖励：预估 × 单价
- 阶梯奖励：保守估计可达到的档位

**示例**：
```
活动A：发布微头条，奖励 10-50 元（按阅读量）
评估：保守估计 20 元
活动B：发布文章，奖励 100 元（固定）
评估：100 元
→ 优先选择活动B
```

### 流量收益估算

```
流量价值 = 曝光量 × 转化率 × 单用户价值
```

**参考标准**：
- 1000 曝光 ≈ 10-50 个新粉丝
- 1 个活跃粉丝 ≈ 0.1-1 元/月长期价值
- 流量扶持时效：通常 3-7 天

**示例**：
```
活动：流量扶持 7 天
评估：预计增加 500 曝光 → 5-25 新粉丝
     → 长期价值 0.5-25 元
```

### 荣誉收益评估

```
荣誉价值 = 可见度提升 + 信任度提升
```

**评估方法**：
- 排行榜前 100：高价值（平台推荐）
- 徽章/证书：低直接价值（信任度提升）
- 创作者等级：中价值（解锁更多权限）

## 成本评估

### 时间成本

```
时间成本 = 内容创作 + 执行操作 + 等待审核
```

| 操作类型 | 预估时间 |
|----------|----------|
| 微头条创作 | 5-15 分钟 |
| 文章创作 | 20-60 分钟 |
| 一键参与 | 1 分钟 |
| 点赞转发 | 1-2 分钟 |
| 填写表单 | 2-5 分钟 |
| 等待审核 | 10-60 分钟 |

### 审核风险成本

```
风险成本 = 失败概率 × 重试时间
```

**风险因素**：
- 内容审核：5-30% 失败率
- 活动规则理解错误：10-20% 失败率
- 技术问题：5-10% 失败率

## 收益成本比计算

### 公式

```
ROI = (预期收益 - 时间成本价值) / 时间成本
```

**时间价值参考**：
- 保守：20 元/小时
- 中等：50 元/小时
- 激进：100 元/小时

### 计算示例

```
活动A：现金 50 元，预计耗时 30 分钟
ROI = (50 - 25) / 0.5 = 50 元/小时

活动B：现金 100 元，预计耗时 2 小时
ROI = (100 - 100) / 2 = 0 元/小时

→ 优先活动A（更高 ROI）
```

## 筛选决策矩阵

| 指标 | 权重 | 评分标准 |
|--------|------|----------|
| 收益金额 | 40% | >100(5) / 50-100(4) / 20-50(3) / <20(1) |
| 时间成本 | 30% | <10min(5) / 10-30min(4) / 30-60min(2) / >60min(1) |
| 成功概率 | 20% | >80%(5) / 60-80%(3) / <60%(1) |
| 紧急程度 | 10% | 即将结束(5) / 还有3天(3) / 充足(1) |

**总分 = 各项得分 × 权重**

### 决策规则

```
总分 > 4.0  → 立即执行
总分 3.0-4.0 → 加入队列
总分 2.0-3.0 → 备选
总分 < 2.0  → 跳过
```

## 快速评估表单

使用 playwright-cli 快速提取活动关键信息：

```bash
playwright-cli eval "
  ({
    title: document.querySelector('.activity-title')?.textContent,
    reward: document.querySelector('.reward')?.textContent,
    deadline: document.querySelector('.deadline')?.textContent,
    requirement: document.querySelector('.requirement')?.textContent,
    type: Array.from(document.querySelectorAll('.tag'))
              .map(t => t.textContent).join(', ')
  })
"
```

**输出示例**：
```json
{
  "title": "科技话题激励活动",
  "reward": "最高50元现金",
  "deadline": "2026-02-20",
  "requirement": "发布带#科技#话题的微头条",
  "type": "生成原创, 现金激励"
}
```

## 批量评估脚本

```bash
# 获取多个活动并批量评分
for activity in $(uv run toutiao-agent activities --limit 10); do
  # 提取信息
  info=$(playwright-cli eval "..." $activity.url)

  # 计算评分（需实现评分逻辑）
  score=$(calculate_score "$info")

  # 输出结果
  echo "$score | $activity.title | $activity.reward"
done | sort -rn  # 按评分排序
```
